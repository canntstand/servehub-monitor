# servehub-monitor
Этот проект представляет собой законченную экосистему для мониторинга микросервисов и автоматизации инфраструктуры. Система не просто запускает сайт, а обеспечивает полный цикл: от авто-деплоя до детальной визуализации метрик и уведомлений в реальном времени.

## Цель проекта

Создать отказоустойчивую и наблюдаемую среду, имитирующую реальный Production-контур и спроектированную так, чтобы минимизировать человеческий фактор при развертывании и обеспечить прозрачность работы каждого узла.

## Как это выглядит

Процесс работы системы выстроен по классической DevOps-цепочке:

1. **Разработка**: Вносятся изменения в код FastAPI или настройки мониторинга, после чего выполняется `git push` в GitLab.
2. **Автоматизация (CI/CD)**: GitLab CI запускает пайплайн. Локальный GitLab Runner подхватывает задачу и вызывает Ansible.
3. **Развертывание**: Ansible автоматически настраивает Docker, обновляет контейнеры с приложением, базой данных PostgreSQL и всеми экспортерами.
4. **Сбор метрик**: Prometheus опрашивает (scrape) сервисы. FastAPI отдает данные о запросах, а экспортеры — о здоровье базы данных и состоянии процессора/памяти.
5. **Визуализация**: В Grafana обновляются дашборды, позволяя наблюдать графики нагрузки и активности приложения в реальном времени.
6. **Удаленный доступ**: Через защищенный туннель предоставляется доступ к API или Grafana из любой точки мира без необходимости иметь «белый» IP.

## Что используется

- **App Layer**: FastAPI (Python), PostgreSQL (Database), SMTP (Email Service).
- **Automation**: GitLab CI/CD (Pipeline), Ansible (IaC — Infrastructure as Code).
- **Monitoring & Logging**: Prometheus (TSDB), Grafana (Dashboards).
- **Exporters**: Postgres Exporter (метрики БД), Node Exporter (метрики железа).
- **Alerting**: Telegram Bot (мгновенные уведомления о сбоях).
- **Infrastructure**: Docker & Docker Compose.
- **Network**: Tunneling solution (безопасный доступ к локальному хосту).

## Основной функционал

- **Автоматизированный деплой**: Полное развертывание всей среды одной командой через Ansible или автоматически при пуше в репозиторий.
- **Умная рассылка**: FastAPI сервис выполняет отправку почтовых уведомлений по заданному расписанию (Cron-tasks).
- **Глубокий мониторинг БД**: Отслеживание медленных запросов, количества подключений и использования памяти PostgreSQL.
- **Система алертинга**: Настройка пороговых значений (например, CPU > 90% или БД недоступна) с отправкой уведомлений в Telegram.
- **Healthchecks**: Мониторинг доступности сервиса; если FastAPI упадет, система зафиксирует простой.
- **External Tunneling**: Возможность демонстрировать работу проекта или подключать внешние вебхуки, работая на локальном компьютере.

## Ключевые фишки

- **Полная автоматизация**: Весь процесс от коммита до работающего продакшена не требует ручных операций на сервере.
- **Единая точка входа**: Один репозиторий содержит и код приложения, и всю конфигурацию мониторинга, и сценарии деплоя.
- **Наблюдаемость из коробки**: Prometheus + Grafana настроены сразу на сбор метрик приложения, базы данных и хоста.
- **Мгновенные уведомления**: Telegram-бот оповещает о критических событиях быстрее, чем кто-либо успеет заметить проблему вручную.
- **Безопасный удаленный доступ**: Туннелирование позволяет демонстрировать работающую систему или принимать вебхуки без публичного IP и настройки роутеров.
- **Отказоустойчивость**: Связка healthchecks + алертинга + автоматического перезапуска контейнеров минимизирует время простоя.
- **Production-готовность**: Все компоненты настроены с учетом лучших практик — лимиты ресурсов, изоляция сетей, персистентные тома.
